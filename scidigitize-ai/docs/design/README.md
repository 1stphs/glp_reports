# GLP-Intelligence Suite 设计文档概览

本目录包含了 **GLP-Intelligence Suite** 的核心设计文档，详细定义了从首页、任务控制中心到具体单元格详情页的完整用户体验和系统逻辑。该系统旨在通过 AI 辅助实现 GLP (Good Laboratory Practice) 报告的自动化生成、合规校验和全流程溯源。

## 文档清单

### 1. 核心页面流程设计

*   **[首页设计.md](首页设计.md)**
    *   **定位**：系统的入口与生产力监控中心。
    *   **核心升级**：
        *   **新建向导 (Wizard)**：分步指派 Agent、模板与合规标准。
        *   **模拟体验区 (Demo Mode)**：即使是新员工也能在沙箱中安全体验“错误修复”。
        *   **生产力看板**：实时展示数据健康度 (Health) 与待办事项。

*   **[任务控制中心页面设计.md](任务控制中心页面设计.md)**
    *   **定位**：单个专题（Study）的指挥部 / Study Control Center。
    *   **核心升级**：
        *   **泳道折叠 (Swimlanes)**：按章节组折叠任务矩阵，降低认知负载。
        *   **例外管理 (Management by Exception)**：通过“批量批准”与“异常高亮”，引导用户只关注红灯项目。

*   **[单元格详情页面设计.md](单元格详情页面设计.md)**
    *   **定位**：任务矩阵中每个单元格的具体工作区。
    *   **核心重构**：**垂直流式页面 (Vertical Progressive Flow)**。
    *   **交互逻辑**：这也是实现“心流”的关键。用户在一个连续的页面中完成 **P1 数据映射 -> P2 事实核对 -> P3 文稿生成** 的全过程，无需频繁跳转。

## 设计核心理念

1.  **渐进式展开 (Progressive Disclosure)**：界面只在用户需要时展示信息。从 P1 到 P3 的自动折叠/展开机制是这一理念的典型应用。
2.  **例外管理 (Manage by Exception)**：通过自动化处理 80% 的“绿色任务”，让科学家专注于 20% 的“红色异常”。
3.  **左右脑协同 (Left-Right Brain Synergy)**：严谨的 logic 计算与自然的 narrative 生成完美融合。
4.  **全链路溯源 (Full Traceability)**：满足 FDA 21 CFR Part 11 的最高审计要求。
