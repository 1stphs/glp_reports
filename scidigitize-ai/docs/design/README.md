# GLP-Intelligence Suite 设计文档概览

本目录包含了 **GLP-Intelligence Suite** 的核心设计文档，详细定义了从首页、任务控制中心到具体单元格详情页的完整用户体验和系统逻辑。该系统旨在通过 AI 辅助实现 GLP (Good Laboratory Practice) 报告的自动化生成、合规校验和全流程溯源。

## 文档清单

### 1. 核心页面流程设计

*   **[首页设计.md](首页设计.md)**
    *   **定位**：系统的入口与生产力监控中心。
    *   **核心内容**：
        *   **专题看板 (Study Dashboard)**：展示所有专题的状态，提供“进入工作台”、“配置”和“历史报告”三个核心入口。
        *   **标准库 (Global Library)**：用于管理公司级的数字资产（SOP、模板、逻辑引擎），实现资产与项目的隔离。
        *   **全局状态**：即时展示待审核数据点、运行中专题等关键指标。

*   **[任务控制中心页面设计.md](任务控制中心页面设计.md)**
    *   **定位**：单个专题（Study）的指挥部 / Study Control Center。
    *   **核心内容**：
        *   采用 **1:3 非对称布局**：左侧为“感知层”（文件清单），右侧为“技能层”（任务矩阵）。
        *   **任务矩阵 (Task Matrix)**：以“章节 x 4阶段 (P1-P4)”的网格形式管理进度。
        *   **任务锁定 (Gating)**：确保只有当前置依赖满足时，后续节点才解锁。

*   **[单元格详情页面设计.md](单元格详情页面设计.md)**
    *   **定位**：任务矩阵中每个单元格的具体工作区，是实现“白盒化 AI”的关键。
    *   **四个核心阶段 (P1-P4)**：
        *   **P1 数据映射 (Data Mapping)**：建立原始文件与数据契约的血缘关系。
        *   **P2 事实核对 (Fact-Check)**：在隔离沙箱中计算并校验数据的准确性 (Logic/Left Brain)。
        *   **P3 文稿生成 (Drafting)**：基于 P2 的事实和标准范例生成报告文本 (Narrative/Right Brain)。
        *   **P4 合规评审 (Sign-off)**：最终预览、合规检查、溯源透视与电子签名。

### 2. 数据与逻辑底层

*   **[GLP 报告自动化逻辑样表 (以生物分析报告为例).md](GLP%20报告自动化逻辑样表%20(以生物分析报告为例).md)**
    *   **定位**：业务逻辑映射表。
    *   **核心内容**：展示了如何将“章节”拆解为“左脑”的结构化数据契约和“右脑”的文本范例。

*   **[生分报告数据契约JSON示例.md](生分报告数据契约JSON示例.md)**
    *   **定位**：前后端交互的数据标准。
    *   **核心内容**：
        *   **Metadata**：计算环境与时间戳。
        *   **Analytical Runs**：具体的分析批次数据（左脑计算结果）。
        *   **Compliance**：内置法规判定的结果（如 Pass/Fail）。
        *   **Audit Trail**：原始文件的溯源信息（Hash, Range）。

## 设计核心理念

1.  **左右脑协同 (Left-Right Brain Synergy)**：严谨的逻辑计算（左脑/Python/Sandboxed）与自然的文本生成（右脑/LLM）分离且协作。
2.  **全链路溯源 (Full Traceability)**：从最终报告的每一个数字都能回溯到原始文件的具体单元格。
3.  **数据契约驱动 (Contract-Driven)**：通过 P1 的数据映射建立稳固的地基，确保后续步骤的可靠性。
4.  **人在回路 (Human-in-the-loop)**：每个关键节点（P1-P4）都设计了人工确认机制，确保合规性。
