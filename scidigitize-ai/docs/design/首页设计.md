# 首页与导航设计 (Homepage & Navigation)

**核心定位**：GLP 实验室的“生产力监控中心 (Productivity Hub)”。

---

## 1. 新建专题向导 (Study Setup Wizard)

为了降低上手门槛并确保合规性，新建专题不再是一个简单的模态框，而是一个**分步向导**。

*   **Step 1: 基础信息 (Basics)**
    *   输入 Study ID (如 `FOXU-2026-BIO-001`)。
    *   选择所属项目 (Project Code)。
*   **Step 2: 资产指派 (Assignment)**
    *   **选择 Agent**：指派哪位数字专家？(如 `BioAnalyst-Pro v2.1`)。
    *   **选择模板**：基于哪个从黄金标准库的模板？(如 `NMPA_PK_Report_Template_v5`)。
*   **Step 3: 合规阈值 (Compliance)**
    *   **选择法规**：`FDA 21 CFR` 或 `NMPA 2020`。此选择将决定 P2 阶段的红线阈值。
    *   **设定剔除规则**：是否允许“技术性剔除”？需要哪一级别的审批？
*   **Step 4: 数据源接入 (Ingestion)**
    *   上传初始 Protocol 和 Raw Data 文件夹。

---

## 2. 模拟体验区 (Simulation Sandbox)

针对新入职员工或演示场景，新增 **"Demo Mode"**。

*   **入口**：首页右下角 floating button "进入模拟实验室"。
*   **内容**：预置了一个 `Demo_Study_001`。
    *   P1 故意有一些字段匹配错误。
    *   P2 故意预埋了 3 个异常值（Outliers）。
    *   P3 故意生成了一段幻觉文本。
*   **目标**：引导用户通过实际操作修复这些“预埋炸弹”，低成本学习系统逻辑。

---

## 3. 专题看板 (Study Dashboard)

### **3.1 状态概览 (Status Header)**

*   **今日待办**：`🔴 3 Critical Alerts` (P2 阶段的严重异常)。
*   **待签发**：`Chanpter 4, 5 (Ready for Sign-off)`。

### **3.2 卡片设计 (Study Cards)**

每一个专题卡片包含：

*   **Header**：Study ID + 进度条 (e.g., "12/15 Sections Locked")。
*   **Health (数据健康度)**：
    *   ✅ **Healthy**：所有数据在阈值内。
    *   ⚠️ **Warning**：存在被剔除的数据点，需关注。
    *   🔴 **Critical**：存在未解释的逻辑矛盾。
*   **Actions (快捷操作)**：
    *   **Workbench** (主按钮)：进入任务控制中心。
    *   **Config** (齿轮)：修改 Agent 或模板（运行中仅允许有限修改）。
    *   **History**：查看审计日志。

## 4. 全局侧边栏 (Global Navigation)

明确区分“项目空间”与“资产空间”。

*   **Workspaces**
    *   🏠 **Home** (所有专题)
    *   📂 **Archives** (已归档专题)
*   **Knowledge Assets (黄金标准库)**
    *   📘 **SOPs & Templates** (报告模板)
    *   🧠 **Logic Engines** (Python 算子库)
    *   📝 **Few-shot Bank** (范例句式库)
*   **System**
    *   🛡️ **Audit Log** (全域审计)
    *   ⚙️ **Settings**
